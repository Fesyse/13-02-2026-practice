### Задача: «Поиск магического числа»

**Условие:** Дан **отсортированный** список чисел. Нужно проверить, содержит ли он число `X`.

---

Представь, что ты работаешь в Google. Твой код должен обрабатывать миллиарды запросов. Если он будет медленным — компания разорится. Реши задачу тремя способами, каждый раз ускоряя её в тысячи раз.

### Вводные данные:

```python
import time

# Создаем огромный отсортированный список
n = 10000
data = list(range(n))
target = 9998  # Число, которое мы ищем
```

---

### Задание 1: "Грубая сила" (Сложность O(n^2))

Напиши самый неэффективный код. Используй вложенный цикл, который зачем-то сравнивает каждый элемент с каждым, чтобы найти `target`.

- **Подсказка:** Проходи циклом по списку, и внутри него еще одним циклом проверяй, не равен ли текущий элемент `target`. Это глупо, но нам нужно увидеть, как это тормозит.

### Задание 2: "Линейный проход" (Сложность O(n))

Напиши обычный поиск.

- **Подсказка:** Используй один цикл `for` или оператор `if target in data`. Это стандартный способ, который мы используем в жизни.

### Задание 3: "Гений поиска" (Сложность O(log n))

Напиши Бинарный поиск.

- **Подсказка:** Дели список пополам на каждом шаге. Это решение должно быть настолько быстрым, что время будет близко к `0.000000`.

## ⏱ Замеряем результаты

start = time.time()
